    Are you new to ClojureScript? Do you need a fast primer?

    This page will teach you how to read Clojure in in XXX minutes. Learning to write Clojure code will take more time, but probably still less than you think.

    ClojureScript is a modern Lisp. Alan Kay once descibed Lisp as "Maxwell's equations of software". Paul Graham has written on how Lisp was a competitive advantage for his startup. For 50 years, the finest minds in computer scinece have refined and polished it.

    Simple Data Literals¶

    Type Example Comment
    character \a The letter a
    number 1
    number 3.14
    string "hello"
    nil nil null, None

    Collection Data Literals¶

    Type Example Comment
    list (1 2) Can be heterogeneous.
    Empty list is ()
    No commas necessary.
    vector [1 2 3] No commas necessary.
    Empty vector is []
    vector [1 "hello" \x nil] Can be heterogeneous.
    map {1 "Hello"} A dictionary, or an associative array.
    map {1 "Hello" 2 nil} No delimiter necessary between pairs.
    map {1 "Hello"
     \b 27
     "pi" 3.14}    		 Can be heterogeneous.
    Still no commas between pairs.

    Collections can nest:

    Example Comment
    vector [1 [2 2] 3] an element can be another vector
    vector [1 {4 5} 3] an element can be a map
    map {1 [4 5]
      2 [\a \b]}    		 a value in a map can be a vector
    map {1 {4 5}
      2 {\a \b}}    		 a value in a map can be a map
    map {{4 5} 1
      [1 2] 2}    		 the keys in a map can be a map or vector

    Symbols¶

    A symbol is a name that is bound to a value.

    Type Example Comment
    symbol inc The symbol inc is bound to one of Clojure's builtin functions.
    This function will return its argument incremented
    symbol + The symbol + is also bound to a builtin function. It adds its arguments.
    note: + is not an operator in ClojureScript, it is a function
    symbol odd? The symbol odd? is also bound to a builtin function. Names in Clojure can include characters which, in other languages, are reserved for operators
    symbol yours You'll soon see how to create your own symbols and bind them to values

    That's It For Syntax¶

    We've now covered almost all of Clojure's syntax. Now we know enough to talk about "evaluation".

    Next, Evaluation¶

    We will now explore how data is evaluated. This is probably an unfamiliar concept for you, right? Evaluating data? What?

    Well, in ClojureScript we evaulate data, to create new data. Dwell on that thought for a minute, it is quite central.

    All data litterals other than lists and symbols evaluate to themselves. So that's really easy.

    Value Evaluates To Comment
    \a \a Same
    1 1 Same
    [1 1] [1 1] Same
    {1 1 2 2 } {1 1 2 2 } Same

    Try it for yourself. Enter an expression into the editor and it will be evaluated. The result will be shown in the box below.

    Evaluating Symbols¶

    Symbols don't evaluate to themselves. Instead, they evaluate to the value they are bound to. (More on how symbols get bound to values very soon)

    Expression Evaluates To Comments
    foo 4 Assuming the symbol foo is bound to the value 4
    bar [1 2 3] Assuming the symbol bar is bound to the value [1 2 3]
    [1 foo] [1 4] Each element is evaluated, and foo evaluated to 4
    [foo bar] [4 [1 2 3]]

    Evaluating Lists¶

    The evaluation of lists is very important to understand.

    Lists evaluate to the return value of a function call, where:

    • the 1st element of the list is assumed to be a function
    • the other elements in the list are assumed to be the actual arguments for the function call
    • the function is called with the arguments
    • the return value of that function is the final evaluation of the list
    List Evaluates To Comments
    (inc 3) 4 First, evaluate each element of the list (there are two):
      • the symbol inc evaluates to a builtin function
      • and 3 evaluates to 3
    Now, we evaluate the list itself which means calling the function
    with the argument. Which is the value 4
    [3 (inc 3)] [3 4]
    (+ 1 2) 3 The symbol + is bound to a builin function.
    And that function adds together its arguments
    (+ 1 2 3) 6 Surprise! Turns out this + function can handle a variable
    number of arguments.
    [1 (+ 1 2 3)] [1 6]

    No operators

    In Clojure, + is a builtin synbol bound to a function. Not an operator.
    Same with -, / , >, =, etc.
    Because there are no operators, there's no operator precidence to worry about.

    Occasionally, you just want a list to evaluate to a list, and not a function call

    List Evaluates To Comments
    (1 1) Error!
    To evaluate a list as a function call, the 1st element must be a
    function.But here it is a number.
    '(1 1) (1 1) No function call happens.
    That leading quote character says to not evaluate the list as
    a function call and to instead just let the list evaluate to itself
    (like other data literals do).
    (list 1 1) (1 1) The builtin function list will make a list from its arguments

    Exercises:

    • is (+ 1) an error? How about (+) or (*)?
    • try (odd? 5)

    Forms¶

    A list like this(+ 1 2) is known as a form.

    Forms can be nested. When they are, evaluation of form elements will involves evaluating child forms.

    Nested Forms Evaluates To Comments
    (+ 3 (inc 1)) 5 Evaluation trace:
      • + is evaluated to a function.
      • 3 is evaluated to 3
      • inc is evaluated to a function.
      • 1 is evaluated to 1
      • (inc 1) is evaluated as a function call which returns 2
      • (+ 3 2) is evaluated as a function call which returns 5
    (= 2 (inc 1)) true = is a symbol too. Also bound to a builtin function

    Try:

    • (inc (dec 1))
    • (odd? (inc (dec 1)))
    • (= (inc (dec 1)) 1)

    Keywords¶

    Keywords are like symbols, excecpt they evaluate to themselves and not to a bound value. This makes them like most other data litterals.

    Keywords are invaluable as identities (eg. keys in maps) and they are used very widely.

    A keyword is a name that starts with a colon.

    Value Evaluates To Comments
    :foo :foo It evaluates to itself.
    :bar :bar It evaluates to itself
    (= + :+) false Is not the same as +, which is a symbol which evaluates
    to a builtin function
    (= :bar :bar) true An important quality. Different instances evaluate to
    equal. Like numbers do, and strings.
    (= :bar :foo) false
    [1 2 :bar] [1 2 :bar] Evaluates like a data litteral
    {1 :bar} {1 :bar}

    Keywords can have a namespace.

    Value Evaluates To Comments
    :panel1/edit :panel1/edit Starts with a colon
    Optionally, contains a /
    Before the / is the namespace
    After the / is the name
    Evaluates to itself
    :panel2/edit :panel2/edit Different namespace to above, same name
    :panel.commands/edit :panel.commands/edit namespaces can be dotted
    (name :a/b) "b" name is a builtin function
    (namespace :a/b) "a" namespace is a builtin function
    (keyword "a/b") :a/b keyword is a builtin function
    (keyword "a" "b") :a/b keyword is a builtin function

    To give you a taste of where this can go, here they are used as the keys in a map:

    {:user/id      1
 :user/name    "Barry"
 :user/age     28
 :user/company "Tesla"}

    Experiments:

    • (namespace :a)
    • (keyword (name :a))
    • (keyword (namespace :a/b) (name :a/b))

    Special Forms¶

    Some Forms are special, and each is special in its own way. Let's review the important ones.

    if¶

    if forms get special treatment.

    The code (if true 4 3) is a four element list. Normal evaluation rules would mean evaluating all four elements of the list and then calling the if function with three arguments.

    Except that isn't what happens. if forms are special, and not all elements are evaluated. The 1st test element is evaluated and then either the 3rd or the 4th argument is evaluated depending on the result of that test. To put that the other way, one of the 3rd or 4th elements will not be evaluated.

    Example Evaluates To Comments
    (if true 4 3) 4 Only true and 4 elements are evaluated
    The 3 element is not evaluated
    (if false 4 3) 3 Only false and 3 elements are evaluated
    (if false 4) nil else form not provided for evaluation, so
    the if evaluates to nil
    (if (odd? 3) 3 4) 3 (odd? 3) evaluates to true
    (= 4 (inc 3)) true
    (if (= 4 (inc 3)) :t :f) :t
    [1 (if true 4)] [1 4]
    {1 (if false 4)} {1 nil}

    Exercises:

    • check if (if true) is valid.
    • what is (if "hello" true false) or (if [] true false) or (if nil true false)
    • when is an alternative to if when there is no else part. Try it out.

    fn¶

    Evaluating an fn form creates an function.

    Here is a very simple example (fn [x] x):

    • as you can see, an fn form has two elements:

      • a vector of symbols - in this case [x]
      • a body - in this case x

    • an fn form is "special" because these two elements are not evaluated.

    • it is only, later, when this function is called that body will be evaluated
    • and, when that evaluation happens, body can refer to symbols in the vector, which will be bound to the actual argument values of the function call
    • our simple example function above only takes one argument [x]
    • and when the body is evaluated, the symbol x will evaluate to whatever x is bound to (that's what symbols do, afterall) and that will be the actual argument
    • the function will return the evaluation of the body, which, in the case above, is the value of the argument supplied.
    • consequently, if we called this function with the argument 3, this function would return 3
    • also, if we called this function with the argument [:a :b], it would return [:a :b]

    Q: how do we call this function?

    A: Just like we call all functions. Place it as the 1st element of a form. Add a 2nd element in the list, which is the actual argument. And then evaluate this list. That's how function calls always happen. So, like this:

    ; a line which starts with a semi-colon is a comment

; The following is a two element list:
;   - the 1st element is a function created by an `fn` form
;   - the 2nd element is "the actual arg"  
((fn [x] x) "the actual arg")

    The function is called with one actual argument: "the actual arg". And the form will evaluate to "the actual arg".

    ((fn [x] x) [:a :b])

    The function is called with one argument [:a :b]. And the form will evaluate to [:a :b].

    How about we create another fucntion: (fn [val] (+ val 1)). The symbol for the actual argument is now val. So that's different. And the body is now the form (+ val 1).

    What if we evaluated it like this:

    ((fn [val] (+ val 1)) 4)

    Try It:

    ((fn []))

    ((fn [x] [x x]) 4)

    or

    ((fn [x] {:a x}) 4)

    or

    ((fn [yes?] {:a (if yes? "yes")}) true)

    what if we called this function with false.

    def¶

    The def form creates a symbol and binds a value to it. Finally, we get to create out own symbols!!

    Example use:

    (def gurus 2)

    This defines the symbol gurus and binds it to the value 2. If, later, you were use this symbol in other code, it would evaluate to 2.

    def is a "special form" on two counts:

    • when evaluated, it adds to the global set of defined symbols and such mutation is known as a side-effect. Functions don't normally cause side effect - they are normally pure.
    • in a normal form for some f, like this (f gurus 2), the gurus element would be evaluated. But not with a def form. It is, instead, the new symbol to define.
    (def skeptics (inc gurus))    ;; using the symbol `gurus` !!

    Now skeptics is a defined symbol and it is bound to the evaluation of (inc gurus) which is the value 3.

    Consider these two:

    (def beach-list [:hat :sunglasses :towel])
(def beach-items (count beach-list))     ;; count is a builtin function

    Now beach-items is a defined symbol and it is bound to 3

    Now, let's take this further and bind a function to a symbol:

    (def my-inc (fn [val] (+ val 1))

    We're creating a function via fn, and we are binding to a symbol using def. Two steps. Using that combination, we've created something similar to the builtin inc.

    (my-inc 4)

    evaluates to 5

    And again:

    (def square-it (fn [x] (* x x)))

    We can use this symbol square-it in a form (it is now bound to a function), like this:

    (square-it 3)

    The value bound to square-it (a function) will be called with the argument 3, which evaluates to the value 9

    defn¶

    Combining def and fn is clumsy. So there is a simpler, one form method called defn. It is special in all the same ways that fn and def are special.

    It is used like this:

    (defn dec      ; `dec` is the symbol being defined
  [val]        ; a vector of symbols for the actual arguments 
  (- val 1))   ; function body - is evaluated when function is called - uses `val`

    This could have been done on one line, but we're now starting to format our code correctly. 2 space indents.

    Used like this:

    (dec 4)

    evaluates to 3

    Define another:

    (defn square-it  ; `square-it` is the symbol being defined
  [x]            ; a vector of symbols for the actual arguments 
  (* x x))       ; function body - is evaluated when function is called - uses `x`

    This binds the symbol square-it to a function.

    Use the symbol in a form:

    (square-it 3)

    which will evaluates to 9.

    Or I could call it like this:

    (square-it (inc 3))

    which evaluates to 16 .

    Define another: 

    (defn greet
  [username]
  (str "Hello " username)))

    str is a builtin function which turns each of its arguments into a string and concatenates them.

    I can call greet like this: 

    (greet "world")

    which will evaluate to "Hello world".

    As you can imagine, in a functional language, creating functions is a big deal. You'll be using defn a lot.

    let¶

    let allows you to create a form with bindings and a body 

    ; let bindings are represented as pairs within a vector
; In this case, only one pair. The symbol `a` is bound to "pen"
(let [a "pen"]   
  a)          ; the evaluation of a `let` form is the evaluation of the body

    This let form evaluates to "pen"

    Another variation:

    (let [a "pen"]   
  :other)         ; oops not even using the bound sysmbol `a`

    This let form evaluates to :other

    Notice the way a let form is formatted (two space indents)

    (let [a  "pen"         ; this pair means `a` is bound to "pen" 
      b  "sword"]      ; this pair means `b` is bound to "sword"
  (> a b))             ; is "pen" greater than "sword" ??

    evaluates to false. Wait, what? That isn't in the story I was told.

    We need to make some changes:

    (let [a  "a pen"
      b  "the sword"]   
  (if (> a b) a b))

    evaluates to "a pen". Phew! Thank goodness for cheep lexograpic hacks which confirm wisdom.

    (let [a  "pen"
      b  "sword"]   
  {:winner (if (> a b) a b)})

    evaluates to {:winner "sword"}

    Often, a let is used within a defn like this:

    (defn items-text 
  [items] 
  (let [num    (count items)
        plural (if (= num 1) "" "s")
        verb   (if (> num 1) "are " "is ")]
    (str "There " verb num " item" plural)))

    Check on the output in these cases:

    (items-text [:towel :sunglasses])

    (items-text [:towel])

    (items-text [])

    Exercise:

    • when there are no items make the text "there are no items" (and not "there is 0 items")

    BuiltIn Functions¶

    Clojure's provides a substantial library of biltin functions. To write Clojure well, you'll have to master this library, but that takes time.

    But in the meantime, to read the Reagent and re-frame tutorials, there's a couple of functions which are essential.

    Along the way, we have introduced a few builtin functions but there are a couple which are used all the time. And we'll need to understand them.

    assoc¶

    assoc allows you to add a key/value pair to a map.

    Example Evaluates To Comments
    (assoc {} :a 4) {:a 4} adding a pair to an empty map {}
    (assoc nil :a 4) {:a 4} nil is treated as {}
    (assoc {:b 1} :a 4) {:b 1 :a 4}
    (assoc {} :b 1 :a 4) {:b 1 :a 4} add two a pairs

    dissoc allows you to remove a key/value.

    map¶

    Example Evaluates To Comments
    (map inc [1 2 3]) (2 3 4) applies inc to each element
    Like [(inc 1) (inc 2) (inc 3)] except result is up with a list
    (map + [1 2 3] [4 5 6]) (5 7 9) applies + to pairs of elements
    Like [(+ 1 4) (+ 2 5) (+ 3 6)], except result is a list
