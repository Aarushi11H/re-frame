    Are you new to ClojureScript? Need a quick primer?

    This page will teach you how to read Clojure in in 20 minutes. Plus a bit of writing. Fully learning to write Clojure code will take more time, but probably still less than you think.

    ClojureScript is a modern Lisp. Alan Kay once descibed Lisp as "Maxwell's equations of software". Paul Graham believed Lisp was a competitive advantage for his startup. For 50 years, some of the finest minds in software have admired it, used it and refined it.

    As the owner of a company which develops software products, I believe ClojureScript gives us a competitive edge. As a programmer, I feel like it has provided me with excellent, nourishing brain food. It is a stable productive place, with little churn. In contrast, I look across to the Javascript landscape and it looks like cruel and unusual punishment.

    So, 20 mins ...

    A symbol is a name that is bound to a value.

    Type Example Comment

    This function will return its argument incremented
    symbol + The symbol + is also bound to a builtin function. It adds its arguments.
    note: + is not an operator in ClojureScript, it is a function
    symbol yours You'll soon see how to create your own symbols and bind them to values

    Evaluation¶

    Time to explore how data is evaluated. Wait. Really? Evaluating data?

    In ClojureScript we evaulate data, to create new data.

    All data litterals other than lists and symbols evaluate to themselves. So that's really easy.

    Value Evaluates To Comment
    \a \a Same
    1 1 Same
    [1 1] [1 1] Same
    {1 1 2 2 } {1 1 2 2 } Same

    Try it for yourself. Enter an expression into the following editor and it will be evaluated. The result will be shown in the box below.

    XXX

    So, let's talk about the two exceptions lists and symbols.

    Evaluating Symbols¶

    Symbols don't evaluate to themselves. Instead, they evaluate to the value to which they are "bound". (More on how symbols get bound to values soon)

    Expression Evaluates To Comments
    foo 4 Assuming the symbol foo is bound to the value 4
    bar [1 2 3] Assuming the symbol bar is bound to the value [1 2 3]
    [1 foo] [1 4] Each element is evaluated, and foo evaluated to 4
    [foo bar] [4 [1 2 3]]

    Evaluating Lists¶

    The evaluation of lists is how function calls occur. In a functional language, function calls are a big deal, so understanding this is important.

    Here's a list (f arg1 arg2 arg3). It has four elements. This list will be evaluated in two steps:

    1. each element in the list is evaluated (all four of them)
    2. a function call occurs
      • the evaluation of the 1st element is the function called
      • the evaluation of the other elements (3 in the case above) become the actual arguments to this function call

    The entire list evaluates to be the return value of the function call.

    Symbols evaluating to functions

    In Clojure code, the 1st element of a list is often a symbol. Eg. (inc 3). inc is a symbol.

    We already know that symbols evaluate to what they are bound to, so in step 1, a symbol, like inc will evaluate to the function it is bound to. And it is that function which will be called in step 2.

    The symbol inc is bound to a function in the standard library. And, perhaps surprisingly for you, the symbol + is also a symbol which is bound to a function in the standard library (which, less surprisingly, adds its arguments together).

    Examples:

    List Evaluates To Comments
    (inc 3) 4 Step 1: evaluate each element of the list (there are two):
      • the symbol inc evaluates to a builtin function
      • and 3 evaluates to 3
    Step 2: call the function with 3 as the argument.
    The return value 4
    [3 (inc 3)] [3 4] Evaluate each element of the vector. Evaluating that 2nd element
    is a function call.
    (+ 1 2) 3 The symbol + is bound to a builin function.
    And that function adds together its arguments
    (+ 1 2 3) 6 Surprise! Turns out this function can handle a variable
    number of arguments.
    [1 (+ 1 2 3)] [1 6]

    No operators

    In Clojure, + is a builtin symbol bound to a function. Not an operator.
    Same with -, / , >, =, etc.
    Because there are no operators, there's no operator precedence to discuss.

    Occasionally, you just want a list to evaluate to a list, and not to a function call

    List Evaluates To Comments
    (1 1) Error!
    A list will be evaluated as function call, and that 1st element will not
    evaluate to a function. It is a number.
    '(1 1) (1 1) No function call happens.
    That leading quote character says to not evaluate the list as
    a function call and to instead just let the list evaluate to itself
    (like other data literals do).
    (list 1 1) (1 1) The builtin function, bound to the symbollist, will make
    a list from its arguments

    Exercises:

    • is (+ 1) an error? How about (+) or (*)?
    • try (odd? 5)

    Forms¶

    A list like this (+ 1 2) is known as a form and forms can be nested like this: (+ 1 (inc 10)).

    In your mind's eye, see this as (f arg1 arg2) where:

    • f is +
    • arg1 is 1
    • arg2 is (inc 10) <-- a nested form

    To evaluate this form, the first step is to evaluate each of the three elements. So the execution trace will be:

    • + will evaluate to the builtin function bound to that symbol
    • 1 evaluates to 1
    • (inc 10) is a form which means we have to apply the same process to its evaluation: (1) evaluate the elements (2) perform the function call
    • with all three elements evaluated, the function is called with the two actual argumetns of 1 and 11
    Nested Forms Evaluates To Comments
    (+ 3 (inc 1)) 5 Evaluation trace:
      • + is evaluated to a function
      • 3 is evaluated to 3
      • (inc 1) is evaluated as a function call which returns 2
      • call function with args 3 and 2, which returns 5
    (= 2 (inc 1)) true = is a symbol which is bound to a builtin function.
    You can guess what it does.
    (= (inc 1) (dec 3)) true dec decrements its arg by one

    Try:

    • (inc (dec 1))
    • (odd? (inc (dec 1)))
    • (= (inc (dec 1)) 1)

    Keywords¶

    Keywords are like symbols, excecpt they evaluate to themselves and not to a bound value. This makes them like most other data litterals.

    Keywords are invaluable as identities (eg. keys in maps) and they are used very widely.

    A keyword is a name that starts with a colon.

    Value Evaluates To Comments
    :foo :foo It evaluates to itself.
    :bar :bar It evaluates to itself
    (= :bar :bar) true Different instances will evaluate to equal. Like numbers
    do, and strings.
    (= :bar :foo) false
    [1 2 :bar] [1 2 :bar] Yep, evaluates to itself.
    {1 :bar} {1 :bar} And again, but as a map value

    Keywords can have a namespace.

    Value Evaluates To Comments
    :panel1/edit :panel1/edit Starts with a colon
    Optionally, contains a /
    Before the / is the namespace
    After the / is the name
    Evaluates to itself
    :panel2/edit :panel2/edit Different namespace to above, same name
    :panel.commands/edit :panel.commands/edit namespaces can be dotted
    (name :a/b) "b" name is a builtin function
    (namespace :a/b) "a" namespace is a builtin function
    (keyword "a/b") :a/b keyword is a builtin function
    (keyword "a" "b") :a/b keyword is a builtin function

    To give you a taste of where this can go, here they are used as the keys in a map:

    {:user/id      1
 :user/name    "Barry"
 :user/age     28
 :user/company "Tesla"}

    Experiments:

    • (namespace :a)
    • (keyword (name :a))
    • (keyword (namespace :a/b) (name :a/b))

    Kebab Case Please¶

    We don't use _ in names. Instead we use -.

    That leads to kebab casing like this :the-winner-is, and not snake casing :the_winner_is. And certainly no camel casing or pascal case like :TheWinnerIs.

    This applies to both symbols and keywords.

    We can use - in names because it isn't an operator. In the form (- 3 2), - is a one character name which is the ultimate in kebab case. It also happens to look like the minus operator in other languages.

    On that subject, you'll often see names which include a trailing ?, like even? or nil?. This is a convension. It is used for predicates which test for thruth.

    Form Evaluates To
    (odd? 5) true
    (even? 5) false
    (nil? 5) false
    (nil? nil) true
    (empty? []) true
    (empty? [:some :thing]) false

    Again, we can use ? in names because it is not an operator, as it is in other lanugages.

    Special Forms¶

    Some Forms are special because they evaluate differently to the "normal" rules outlined above. Let's review the important ones.

    if¶

    if forms evaluate in a "special" way.

    This (if true 4 3) is a four element list. Normal evaluation rules would mean, first, evaluate all four elements of the list, and then calling the if function with three arguments.

    But with if forms, not all elements are evaluated. The 2nd test element is evaluated but then either the 3rd or the 4th argument is evaluated depending on the result of that test. One element remains unevaluated.

    Example Evaluates To Comments
    (if true 4 3) 4 Only true and 4 elements are evaluated
    The 3 element is not evaluated
    (if false 4 3) 3 Only false and 3 elements are evaluated
    (if false 4) nil else form not provided for evaluation, so
    the if evaluates to nil
    (if (odd? 3) 3 4) 3 (odd? 3) evaluates to true
    (= 4 (inc 3)) true
    (if (= 4 (inc 3)) :t :f) :t
    [1 (if true "yes")] [1 "yes"]
    {1 (if false "yes")} {1 nil}

    Exercises:

    • check if (if true) is valid.
    • what is (if "hello" true false) or (if [] true false) or (if nil true false)

    fn¶

    An fn form creates an function.

    Here is a very simple example (fn [x] x):

    • an fn form has three elements:
      • the 2nd is a vector of symbols - in this case [x]
      • the 3rd is a body - in this case x
    • an fn form is "special" because the 2nd and 3rd elements are not evaluated
    • it is only later, when this function is called, that body will be evaluated, and when that happens, body can refer to symbols in the vector, which will be bound to the actual arguments of the function call
    • our simple example function above only takes one argument [x]
    • and, when the body is evaluated, the symbol x will be bound to the actual argument of the function call
    • the function will return the evaluation of the body, which, in the case above, is the same value as the actual argument.
    • consequently, if we called this function with the argument 3, this function would return 3
    • and, if we called this function with the argument [:a :b], it would return [:a :b]

    Question: so, how do we call this function?

    Answer: place it as the first element in a form. Then add a 2nd element in that form, which is the actual argument. Like this:

    ; Aside: a line which starts with a semi-colon is a comment

; The following is a two element list:
;   - the 1st element is a function created by an `fn` form
;   - the 2nd element is "the actual arg"  
((fn [x] x) "the actual arg")

    This calls the function with one actual argument: "the actual arg". This form will evaluate to "the actual arg".

    Another call:

    ((fn [x] x) [:a :b])

    The actual argument is [:a :b], and the form (function call) will evaluate to [:a :b].

    Let's create a different function and evaluate it: 

    ((fn [num] (+ num 1)) 4)

    The body of the function, (+ num 1), will be evaluated with the num symbol bound to the actual parameter 4. The return value is 5.

    Try:

    ((fn [x] [x x]) 4)

    or

    ((fn [x] {:a x}) 4)

    or

    ((fn []))

    or

    ((fn [yes?] {:a (if yes? "yes")}) true)

    what if, instead, we called this function with false.

    You won't see this written in a ClojureScript code, because it is wierd, but here's a puzzle:

    (((fn  [] inc)) 4)

    What is the evaluation? Note: there is an extra set of parens around the fn form.

    def¶

    The def form creates a symbol and binds it to a value. 

    (def gurus 2)

    This defines the symbol gurus and binds it to the value 2. If, later, we used this symbol in other code, it would evaluate to 2.

    def is a "special form" in two ways:

    • when evaluated, it adds to the global set of defined symbols. Such mutation is known as a side-effect. Functions don't normally cause side effect - they are normally pure.
    • in a normal form for some f, like this (f gurus 2), the gurus element would be evaluated. But the evaluation rules in a def form are different because that's the symbol to define. It doesn't get evaluated. But the 3rd element certainly does.

    The 3rd element of a def is evaluated: 

    (def skeptics (inc gurus))    ;; using the symbol `gurus` !!

    skeptics is now also a defined symbol and it is bound to the evaluation of (inc gurus), which is (inc 2), which is 3.

    Consider these two:

    (def beach-list [:hat :sunglasses :towel])
(def beach-items (count beach-list))     ;; count is a builtin function

    beach-items is a symbol bound to 3

    Now, consider what is happening here:

    (def my-inc (fn [a] (+ a 1))

    That fn form will create a function, and then the def will bind it to the symbol my-inc. Two steps. Hey, we've just created our own inc.

    (my-inc 4)

    evaluates to 5

    And again:

    (def square-it (fn [x] (* x x)))

    We can use this symbol square-it in a form (it is now bound to a function), like this:

    (square-it 5)

    evaluates to 25

    defn¶

    Creating a function happens a lot, and combining def and fn each time would be verbose. So, there is a terser method which combines the two, called defn.

    It is used like this:

    (defn dec      ; `dec` is the symbol being defined
  [n]          ; a vector of symbols for the actual arguments 
  (- n 1))     ; function body - evaluated when function is called - uses `n`

    This could have been done on one line, but we're now starting to format our code correctly. 2 space indents.

    Use it: 

    (dec 4)

    evaluates to 3

    Define another function:

    (defn square-it  ; `square-it` is the symbol being defined
  [n]            ; a vector of symbols for the actual arguments 
  (* n n))       ; function body - evaluated when function is called - uses `n`

    use it: 

    (square-it 3)

    evaluates to 9.

    Or use it like this:

    (square-it (inc 3))

    evaluates to 16 .

    Define another: 

    (defn greet
  [username]
  (str "Hello " username)))

    str is a builtin function which turns each of its arguments into a string and concatenates them.

    use it: 

    (greet "world")

    evaluates to "Hello world".

    In a functional language, creating functions is a big deal. You'll be using defn a lot.

    let¶

    Here's another special form you'll be using a lot. let allows you to create a form with two parts: bindings and a body 

    ; a let form has two parts: 
;   - 1st a vector of "binding pairs"
;   - 2nd the body 
(let [a "pen"]   ; a pair - binds the symbol `a` to the value "pen"
  a)             ; the body is evaluated using the bindings

    This let form evaluates to "pen"

    Another one: 

    (let [a "pen"]   
  :foo)         ; odd. The body doesn't use `a`. But this still works.

    This let form evaluates to :foo

    Notice the way this let form is formatted (two space indents)

    (let [a  "pen"         ; this pair means `a` is bound to "pen" 
      b  "sword"]      ; this pair means `b` is bound to "sword"
  (> a b))             ; is "pen" greater than "sword" ??

    evaluates to false. Wait, no. That isn't convential wisdom. Damn you lexographic comparisons.

    We need to make some changes:

    (let [a  "the pen"
      b  "a sword"]   
  (if (> a b) a b))

    evaluates to "a pen". Phew! 

    (let [a  "the pen"
      b  "a sword"]   
  {:the-winner-is (if (> a b) a b)})

    evaluates to {:the-winner-is "the pen"}

    let is often used within a defn: 

    (defn items-text 
  [items] 
  (let [num    (count items)
        plural (if (= num 1) "" "s")
        verb   (if (> num 1) "are " "is ")]
    (str "There " verb num " item" plural)))

    Check on the output in these cases:

    (items-text [:towel :sunglasses])

    (items-text [:towel])

    (items-text [])

    Exercise:

    • when there are no items make the text "there are no items" (and not "there is 0 items")

    BuiltIn Functions¶

    Clojure's provides a substantial library of biltin functions. There's a couple of functions which are essential.

    assoc¶

    assoc allows you to add a key/value pair to a map.

    Example Evaluates To Comments
    (assoc {} :a 4) {:a 4} adding a pair to an empty map {}
    (assoc nil :a 4) {:a 4} nil is treated as {}
    (assoc {:b 1} :a 4) {:b 1 :a 4} adding a pair to non-empty map
    (assoc {} :b 1 :a 4) {:b 1 :a 4} add two a pairs

    dissoc allows you to remove a key/value.

    map¶

    Functions that take other functions as arguments are called higher-order functions. map is a higher order function.

    It applies a function to all the elements in a collection.

    Imagine we create a function called plus-one

    (defn plus-one 
  [it]
  (+ 1 it))
    Example Evaluates To Comments
    (map plus-one [1 2 3]) (2 4 6) Applies plus-one to each element of the collection.
    Think [(plus-one 1) (plus-one 2) (plus-one 3)]
    except the result is a list, not a vector.
    (map inc [1 2 3]) (2 4 6) Same as above. But now using the builtin function inc

    Let's create a function called add

    (defn add
  [a b]
  (+ a b))
    Example Evaluates To Comments
    (map add [1 2 3] [4 5 6]) (5 7 9) Applies add to pairs of elements.
    Think [(add 1 4) (add 2 5) (add 3 6)], except
    result is a list
    (map + [1 2 3] [4 5 6]) (5 7 9) Same as above. But using the builtin function +
    (map :a [{:a 1} {:a 11}]) (1 11) XXX explain

    What would this evaluate to:

    (map str (map inc [1 2 3]))

    reduce¶

    reduce is the 600 pound gorilla in the functional world.

    It takes three arguments:

    • a function which "accumulates"
    • the initial state of the accumulation
    • a collection to accumulate "across"
    (reduce +          ;; accumulating with `+` 
        0          ;; initial state of accumulation 
        [1 2 4])   ;; the collection to accumulate across, element by element

    An accumulation function, + above, must accept two arguments:

    • the current, running accumulation, up to the current element
    • the new element

    The accumulation function should combine the running "total" with the new value, and return the new accumulation. + works because it can take two arguments and it will return a new accumulation of the two.

    Because there are three elements in the collection, there are three steps in the reduction.

    This is effectively what reduce will compute for the example above: 

    (+ (+ (+ 0 1) 2) 4)

    evaluates to 7 and in this case reduce is used to sum the collection

    Or, to explain it step by step:

    Collection Element Running
    Accumulation    		 Evaluation
    using +    		 New Accumulation Comment
    1 0 (+ 0 1) 1 + is applied to the initial value 0 and the 1st element 1, producing a new accumulation of 1
    2 1 (+ 1 2) 3 + is applied to the running accumulation 1 and the 2nd element 2 , producing a new accumulation of 3
    4 3 (+ 3 4) 7 + is applied to the running accumulation 3 and the 3rd element 4, producing a new accumulation of 7

    Next example: imagine we create this accumulation function:

    (defn max
  [a b]
  (if (> a b) a b))

    And then we use that function with a reduce:

    (reduce max        ;; accumulating with `max` 
        0          ;; initial accumulation  (maximum)
        [1 2 4])   ;; the collection to process

    (max (max (max 0 1) 2) 4)

    How about this one:

    (reduce str        ;; accumulating with `str` 
        0          
        [1 2 4])

    We "accumulate" a string

    How about this one:

    (reduce conj        ;; <---  `conj` 
        []          ;; <--- empty vector
        [1 2 4])
